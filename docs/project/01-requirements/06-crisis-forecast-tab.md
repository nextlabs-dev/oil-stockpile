# 有事終息予想タブ（/forecast/）要件・仕様

**ステータス**: 仕様確定（2026-07-21）／未実装
**決裁**: 亀井（NextLab 代表）
**関連**: 提言タブ `/opinions/`（#109 で実装済み）

---

## 1. 背景と位置づけ

ホルムズ海峡の通航支障を受け、経済産業省は 2026-03-16 に民間備蓄義務量 15 日分の引き下げと
国家備蓄放出、2026-04-24 に第 2 弾（約 20 日分）の放出を決定している。
「有事がいつ終息するか」は本サイトの本題（備蓄はあと何日もつか）と直結する論点。

本タブは**来訪者の予想を集計して可視化する世論パート**であり、
サイト運営者が地政学的な予測を提示するものではない。

### 元 AI Studio 版からの最重要変更

元 `IranUsMarket.tsx` は以下がすべて**モックだった**ため、そのまま移植しない。

| 元実装 | 問題 |
|---|---|
| YES/NO オッズ（$0.28 等）を `setInterval` の疑似ランダムウォークで変動 | 実市場データでないのに「世界の予測市場のオッズ」と表示していた |
| 投票数 318/753 をハードコード初期値から加算 | 共有カウントに見えるが実体なし |
| `PROBABILITY_HISTORY` 定数の確率推移チャート | 履歴も作り物 |
| 投票は `localStorage` のみ | 集計されない |

**方針: 表示する数値は「自サイトの実投票結果」だけにする。外部オッズは使わない。**

## 2. 確定仕様

| 項目 | 決定 |
|---|---|
| 数値の出所 | **来訪者投票の実結果のみ**（外部予測市場・報道オッズは使用しない） |
| 投票機能 | **あり**。「あなたの予測」＋「みんなの予測」を表示 |
| 集計 | **バックエンドで実集計**（クライアント限定の見せかけ集計は不可） |
| 設問形式 | **期間別の選択肢** |
| バックエンド基盤 | **Cloudflare Workers + D1** |
| 不正対策 | **Cloudflare Turnstile ＋ IP レート制限** |

## 3. 設問設計

> **Q. ホルムズ海峡の通航支障による有事は、いつ終息すると思いますか？**

| id | 選択肢 |
|---|---|
| `m3` | 〜3ヶ月 |
| `m6` | 〜6ヶ月 |
| `y1` | 〜1年 |
| `long` | 1年以上（長期化） |

- 「終息」の定義を設問脇に明記する（例: ホルムズ海峡の通航が平常化し、国家備蓄の追加放出が不要になる状態）。
  定義を書かない設問は解釈がぶれるため必須。
- 設問は当面この 1 問のみ。将来増やす場合も `question_id` で分離する。

## 4. 画面仕様

### 投票前
- 設問と 4 択を提示。**全体結果は伏せる**（先入観・バンドワゴン効果を避けるため）。
- 総投票数のみ表示可。

### 投票後
- **あなたの予測**: 選んだ肢をハイライト表示。
- **みんなの予測**: 選択肢ごとの得票率バー＋実数＋総投票数。
- **サイト本題との接続**: 現在の備蓄残日数（`snapshots.json` の SSOT）と突き合わせ、
  「①〜④のケースで備蓄が足りるか」を併記する。ここが他の予測サイトにない本サイト固有の価値。
- 再投票は不可（変更したい場合の扱いは §8 の未決事項）。

### 表記
- 「参考値・非公式」「本サイト来訪者の回答であり、統計的な世論調査ではない」旨を明記。

## 5. データモデル（D1）

```sql
CREATE TABLE votes (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  question_id   TEXT NOT NULL,           -- 例: 'hormuz_end_2026'
  choice        TEXT NOT NULL,           -- 'm3' | 'm6' | 'y1' | 'long'
  voter_hash    TEXT NOT NULL,           -- IP+UA+日次ソルトの SHA-256（生IPは保存しない）
  created_at    TEXT NOT NULL            -- ISO8601 (UTC)
);
CREATE INDEX idx_votes_q ON votes(question_id);
CREATE UNIQUE INDEX idx_votes_dedup ON votes(question_id, voter_hash);
```

- **生 IP は保存しない**。`voter_hash` にソルト付きハッシュのみ保持（プライバシー方針）。
- ソルトは日次ローテーション（長期の名寄せを不可能にする）。
- 個人情報・Cookie による追跡は行わない。

### 集計
- 都度 `COUNT` で足りる規模だが、閲覧が増えたら集計結果をキャッシュ（KV, 60 秒程度）。

## 6. API 設計（Cloudflare Workers）

別ホスト（例: `api.oilstock.nextlabs.jp`）で提供。GitHub Pages 本体は静的のまま維持する。

| メソッド | パス | 内容 |
|---|---|---|
| `GET` | `/v1/forecast/results?q=<question_id>` | 選択肢ごとの得票数・総数を返す |
| `POST` | `/v1/forecast/vote` | `{question_id, choice, turnstile_token}` を受けて 1 票記録し、最新結果を返す |

- CORS: `Access-Control-Allow-Origin: https://oilstock.nextlabs.jp` に限定。
- レスポンスに個人識別情報を含めない。
- エラー時はフロントで「現在集計を取得できません」を表示し、**推測値やモックにフォールバックしない**。

## 7. 不正対策・セキュリティ

1. **Cloudflare Turnstile**: 投票時にトークン検証。bot を弾く。
2. **IP レート制限**: 同一 `voter_hash` は 1 票（UNIQUE 制約）。加えて Workers 側で短時間の連投を制限。
3. **`localStorage`** は UX 補助（投票済み表示）のみ。**信頼の根拠にはしない**。
4. **CSP 追加が必要**（`build_csp()` を更新）:
   - `connect-src` に Workers の API オリジンを追加
   - `frame-src` / `script-src` に Turnstile（`https://challenges.cloudflare.com`）を追加
5. 監査: 異常な投票増加を検知できるよう `created_at` を保持。

## 8. SEO / AEO 方針

- 投票率は**変動値なので JSON-LD には入れない**（陳腐化・誤引用リスク）。
- 構造化データは他タブと同様に `WebPage` + `BreadcrumbList`（`isPartOf` → `#app`、
  `publisher` → `nextlabs.jp#organization`）に留める。
- 設問と「終息」の定義は `FAQPage` 化を検討可（定義は変動しないため安全）。
- `sitemap.xml` に `/forecast/` を追加。
