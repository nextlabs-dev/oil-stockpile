# **3\. OGP動的生成（v2.0）**

## **3.1 設計変更**

v1.0の3バリアント（neutral/caution/positive）を廃止し、1バリアントに統合。タンクゲージをOGP画像に含めることで、シェア時の視覚的インパクトを最大化する。

## **3.2 OGP画像構成（1200x630px）**

レイアウト:

* 背景: \#fafafa（白）  
* 左半分: 「日本の石油備蓄、いまXXX日分。」+ サイトURL  
* 右半分: タンクゲージ（緑→黄→赤で残量表示）  
* フッター: 「oil-stockpile.jp」小さく表示

## **3.3 エンドポイント**

/api/og?v={version}

@vercel/ogでEdge Runtime生成。vパラメータ \= 月次更新UUIDでCDNキャッシュを自動無効化。

## **3.4 シェアテンプレート**

シェアボタン押下時のテキスト:

*「日本の石油備蓄、いまXXX日分。」+ URL*

XのシェアテキストにはOGP画像が自動展開される。

# **4\. タンカーセクション**

## **4.1 データ取得**

Datalastic AIS APIのLocation Trafficエンドポイントを使用。

* 取得頻度: 1日×1回（毎朝6:00 JST）  
* 対象港: 横浜(JPYOK)・千葉(JPCHB)・川崎(JPKWZ)・四日市(JPYKK)・堰泉北(JPSKT)  
* フィルタ: vessel type \= Tanker、laden（積載中）のみ  
* バレル推計: DWTから算出（VLCC≈200万bbl、Suezmax≈100万bbl）  
* コスト: 月£199（Starterプラン、20,000クレジット）で十分

## **4.2 表示項目**

* 日本向けタンカー隻数  
* 積載量（推計百万bbl）  
* リットル換算  
* 最短到着予定  
* 主な出発地

## **4.3 Cronジョブ**

// vercel.json

{ "crons": \[{ "path": "/api/cron/fetch-tankers", "schedule": "0 21 \* \* \*" }\] }

UTC 21:00 \= JST 06:00。結果をSupabase tanker\_snapshotsテーブルに保存。

# **5\. 石油のものさし**

「バレル」や「万kL」を身近なモノに換算するお役立ちコンテンツ。3タブ構成:

### **① 単位換算**

| 換算 | 説明 |
| :---- | :---- |
| 1バレル \= 159L | お風呂（200L）の約0.8杯分。灯油ポリタンク約9個分。 |
| barrel \= 樽 | 1860年代、米国で原油を酒樽に詰めて運んだ名残。 |
| 日本ではkL | 国際取引はバレル、国内はkL。1kL ≈ 6.29バレル。 |
| 1bbl \= 42ガロン | 米液量ガロン（約3.785L）基準。 |

### **② スケール感**

| 項目 | 数値 | 説明 |
| :---- | :---- | :---- |
| VLCC 1隻 | 200万バレル | 25mプール約1,280杯分 |
| 日本の1日の消費 | 314万bbl/日 | VLCC約1.5隻分。世界第5位 |
| 215日分の備蓄 | VLCC約340隻分 | 約6.7億バレル |
| 中東→日本 | 20〜25日 | VLCCで約3週間の航海 |

### **③ 豆知識**

* なぜ42ガロン？ → 50ガロンの酒樽から漏れて届いた量が42ガロン  
* 日本は輸入97% → 海上輸送依存度が主要国で突出  
* ガソリン1Lの内訳 → 約4割が原油、約6割が税金等  
* 備蓄の歴史 → 1973年オイルショックの教訓から1975年開始
